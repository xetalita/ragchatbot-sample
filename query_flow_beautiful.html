Processing Flow Timeline

1

User Input Capture

JavaScript captures query, disables input, shows loading animation

  • ‚úì User types query in chat input field
  • ‚úì Press Enter or click Send button triggers sendMessage()
  • ‚úì Input field value extracted and trimmed
  • ‚úì Input/button disabled to prevent duplicates
  • ‚úì User message added to chat display
  • ‚úì Loading animation appears
2

API Processing

FastAPI validates request, manages session, routes to RAG system

  • ‚úì POST request sent to /api/query endpoint
  • ‚úì JSON payload contains query and session_id
  • ‚úì FastAPI validates request with Pydantic model
  • ‚úì Check if session_id exists or create new
  • ‚úì Pass request to rag_system.query()
  • ‚úì Handle errors with HTTPException if needed
3

AI Analysis

Claude determines if course search is needed based on query

  • ‚úì Format query into AI prompt with instructions
  • ‚úì Retrieve conversation history from session
  • ‚úì Include available tools (search_course_content)
  • ‚úì Send to Claude API with temperature=0
  • ‚úì Claude analyzes query intent
  • ‚úì Decides: use search tool or respond directly
4

Vector Search

ChromaDB performs semantic search on course embeddings

  • ‚úì Tool manager executes CourseSearchTool
  • ‚úì Optional course name resolution if provided
  • ‚úì Query converted to embeddings via SentenceTransformer
  • ‚úì ChromaDB searches course_content collection
  • ‚úì Apply filters (course name, lesson number)
  • ‚úì Return top K results with metadata & distances
5

Response Generation

Claude synthesizes final answer with context and sources

  • ‚úì Tool results passed back to Claude
  • ‚úì Claude processes search results as context
  • ‚úì Generate comprehensive answer (max 800 tokens)
  • ‚úì Session history updated with Q&A pair
  • ‚úì Extract sources from tool manager
  • ‚úì Format response with answer and sources
6

  • ‚úì Frontend receives JSON response
  • ‚úì Remove loading animation
  • ‚úì Parse markdown in answer using marked.js
  • ‚úì Add message to chat with assistant styling
  • ‚úì Create collapsible sources section if present
  • ‚úì Re-enable input field and focus
